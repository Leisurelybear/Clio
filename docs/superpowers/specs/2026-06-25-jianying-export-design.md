# N-01: JianYing (剪映) Draft Export

## Background

The "zero-editing" (零剪辑) goal requires that plan output be directly importable into JianYing Pro (剪映专业版) without manual timeline setup. Currently, `plan.json` outputs only a structured sequence that users must manually translate into the editing software.

This feature generates a complete JianYing draft (`draft_content.json` + `draft_meta_info.json`) from `plan.json`, producing a ready-to-open project in the editing software.

## Architecture

```
plan.json  →  export_plan_to_jianying()  →  output/export/<day>_jianying/
                                               └── draft_content.json
                                                    (draft_meta_info.json auto-generated by 剪映 on open)
```

New module:

```
vlog_tool/export/
├── __init__.py          →  FORMAT_REGISTRY dict, export() dispatch
└── jianying.py          →  JianYing draft builder (core)
```

`__init__.py` registers formats so `main.py export --format <name>` can dispatch:

```python
FORMAT_REGISTRY: dict[str, Callable] = {
    "jianying": export_plan_to_jianying,
    # "capcut": ...,  # future
    # "premiere": ..., # future
}
```

## Core Mapping: `plan.sequence[]` → `draft_content.json`

### Top-level shape

```python
{
    "id": str(uuid4()),
    "name": plan_data.get("day_title", day_label),
    "duration": total_duration_us,
    "fps": 30,
    "canvas_config": {
        "width": 1920, "height": 1080,
        "ratio": 1.7777777777777777   # 16:9
    },
    "platform": {
        "app_source": "lv",           # JianYing (国内版)
        "app_version": "5.9.0",
        "os": "windows"
    },
    "materials": { "videos": [...], "texts": [...] },
    "tracks": [...]
}
```

### Materials — Videos

Each unique `sequence[].index` resolves to one `material` entry.

Resolution logic (`_resolve_material`):
1. Scan `input_dir` recursively for `{index}_*` pattern
2. Match by stem prefix (e.g. `001` → `001_GL010683.mp4`)
3. Read duration via `get_duration_sec()` (reused from `utils.py`), convert to microseconds
4. If not found, emit warning and skip dependent segments

```python
{
    "id": "a1b2c3d4-...",          # UUID
    "type": "video",
    "path": r"G:\videos\GL010683.mp4",   # absolute path to original
    "duration": 45000000            # microseconds
}
```

### Materials — Texts

Each `sequence[].voiceover_hint` generates one text material entry with double-encoded content.

```python
{
    "id": text_material_uuid,
    "content": json.dumps({
        "text": voiceover_text,
        "font_color": "#FFFFFF",
        "font_size": 18,
        "bold": False,
    })
}
```

### Tracks — Video Track

Each `sequence[]` entry → one segment on the video track.

```python
{
    "id": segment_uuid,
    "material_id": referenced_video_material_uuid,
    "target_timerange": {
        "start": accumulated_start_us,     # timeline position (cumulative)
        "duration": segment_duration_us
    },
    "source_timerange": {
        "start": parse_time_range(use_timeline).start * 1_000_000,
        "duration": segment_duration_us
    }
}
```

### Tracks — Text Track

One text segment per sequence entry, timed to match the video segment:

```python
{
    "id": text_segment_uuid,
    "material_id": referenced_text_material_uuid,
    "target_timerange": {
        "start": same_accumulated_start_us,
        "duration": same_segment_duration_us
    }
}
```

### Time Conversion

```
"00:10-00:45"  →  start=10s, end=45s
                →  source_start=10_000_000μs
                →  duration=35_000_000μs
```

Reuse `parse_time_range()` from `cut.py` (already handles `MM:SS-MM:SS` and `HH:MM:SS-HH:MM:SS`).

### Cumulative Timeline Position

Sequential: each segment's `target_timerange.start` = previous segment's `target_timerange.start + duration`.

## API

### Python

```python
def export_plan_to_jianying(
    plan_path: Path,
    output_dir: Path,
    input_dir: Path,
    day_label: str = "day1",
) -> Path:
    """Generate JianYing draft from plan JSON.

    Returns path to the output draft directory.
    """
```

### CLI (`main.py`)

```python
@cli.command()
@click.option("--format", default="jianying", help="Export format")
@click.option("--day", default="day1")
@click.option("--output", default=None, help="Output directory override")
def export(format, day, output):
    """Export plan to video editing software draft."""
```

Default output: `output/export/<day>_<format>/`

### UI

- Button in plan view (next to "Save"): "导出到剪映"
- `POST /api/export` → `{ok: bool, path: string, skipped: int}`
- Toast notification on success

## Error Handling

| Scenario | Action |
|---|---|
| plan JSON not found | Print error, exit(1) |
| plan.sequence empty | Generate empty draft (skeleton only) |
| Video material not found | Skip segment, warn, continue |
| `use_timeline` parse failure | Skip segment, warn, continue |
| ffprobe fails on video | Skip segment, warn, continue |
| Output dir write failure | Catch IOError, print error, exit(1) |
| Output dir exists | Overwrite (default) |

## Future Compatibility

The `FORMAT_REGISTRY` pattern allows adding new formats without modifying existing code:

```python
# vlog_tool/export/capcut.py
def export_plan_to_capcut(...): ...

# vlog_tool/export/__init__.py
FORMAT_REGISTRY["capcut"] = export_plan_to_capcut
```

## Non-Goals (MVP)

- No video file copying (references originals by absolute path)
- No transitions, effects, or keyframes
- No background music
- No thumbnail generation
- No `draft_meta_info.json` generation (剪映 auto-generates on open)
