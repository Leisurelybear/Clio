# 2026-07-06 Unit Test Review

## Scope

Reviewed the current pytest and frontend unit-test suite after CR-008.

- Python tests collected: 1089
- Backend verification command: `python -m pytest clio/tests/ -q`
- Frontend test command checked: `npm test -- --run clio/ui/static/src/__tests__/runner.test.js`

## Findings Fixed

### UT-001: Transcribe tests leaked `/tmp` state

`clio/tests/test_tasks_transcribe.py` used hard-coded `/tmp/output` and `/tmp/input` in the shared `cfg` fixture. Some tests then changed `cfg.paths.output_dir` without updating `cfg.transcripts_dir`, causing sandbox-only failures and possible dependence on stale host `/tmp` directories.

Resolution:
- The fixture now derives default paths from `tmp_path`.
- Tests that change output paths also keep `cfg.transcripts_dir` in sync.

### UT-002: Trip context test did not test behavior

`test_trip_context_file_loaded` created files and monkeypatches but never called `_wrap_with_context()` or asserted anything.

Resolution:
- The test now creates a real project-level `templates/trip_context.md`, calls `_wrap_with_context()`, and asserts both trip context and prompt are present.

### UT-003: Plan filtering tests duplicated production logic

Two `plan_daily_vlog` tests copied the filtering algorithm into the test body instead of invoking production code. They could pass even if `plan_daily_vlog()` regressed.

Resolution:
- The tests now mock the AI response and call `plan_daily_vlog()` directly.

### UT-004: Duplicate-run progress protection test was ineffective

`test_already_running_does_not_clobber_progress` did not create a progress file, so it could not detect progress clobbering.

Resolution:
- The test now creates `.progress.json` and asserts its content is unchanged after a duplicate start request.

### UT-005: No-assert smoke tests were made explicit

Several tests relied only on "does not raise" behavior.

Resolution:
- Added concrete assertions to config validation, config cache invalidation, logging teardown, progress concurrent-read simulation, and refine busy-state discard coverage.

## Remaining Known Risk

### Frontend unit tests require newer Node

Local Node is `v16.14.0`, but `package.json` requires `>=18`, and Vitest/Vite currently fails during startup with:

```text
node:fs/promises does not provide an export named constants
```

The frontend test itself is meaningful, but it cannot be used as a local gate until Node is upgraded. CI should continue using a supported Node version.

### Route dispatch tests are intentionally mock-heavy

Many `clio/tests/test_server.py` route-dispatch tests assert that the correct handler is called. These are useful for route table regressions, but they do not prove handler behavior by themselves. Handler-level tests should remain the main behavior coverage.
